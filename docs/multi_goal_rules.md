# Multi Goal Rules


---

## MVP ограничение

В MVP версии у пользователя может быть только одна активная цель.

Нельзя создать новую цель, пока текущая цель:
- не завершена (completed)
- или не отменена (cancelled)

Переключение между целями отсутствует.

---

## 4. Отдельный контекст для каждой цели

Каждая цель должна иметь собственный набор данных.

Для каждой цели отдельно существуют:
- `goal_id`
- `goal_session`
- `plan`
- `plan_steps`
- `daily_checkins`
- `proof_files`
- `goal_progress`
- `weekly_summary`

Backend не должен работать с логикой "текущие данные пользователя вообще".
Backend должен работать по схеме:
- пользователь
- конкретная цель
- действие по этой цели

То есть почти каждое действие должно быть привязано к `goal_id`.


---

## 6. User Chat Context

Отдельно существует глобальный chat context пользователя.

Он хранит:
- `user_id`
- `active_goal_id`
- `last_selected_goal_id`
- глобальный state чата при необходимости
- updated_at

Это нужно только для навигации внутри одного чата.
