# Product Flow

## 1. Start
Пользователь нажимает /start.

## 2. Goal creation
Бот спрашивает, какой цели пользователь хочет достичь.

## 3. Goal clarification
После описания цели бот задает уточняющие вопросы:
- текущий уровень
- ограничения
- доступные ресурсы
- мотивация
- сколько времени в день/неделю готов уделять

## 4. Plan generation
На основе ответов backend запрашивает у AI план достижения цели.

## 5. Plan review
Бот показывает пользователю план.
Пользователь может:
- принять план
- попросить скорректировать план

## 6. Goal execution
После принятия плана цель становится активной.
Пользователь начинает выполнять шаги плана.

## 7. Daily proof
Для подтверждения выполнения пользователь присылает фотопруф или скриншот.

## 8. Proof validation
Система проверяет, соответствует ли изображение выполнению нужного шага.
Результат:
- accepted
- rejected
- uncertain

## 9. Daily progress
Каждый день бот отправляет прогресс по цели в процентах.

## 10. Weekly summary
Каждую неделю бот отправляет summary по прогрессу и мотивационное сообщение.

## 11. Multi-goal mode
У пользователя может быть несколько целей.
У каждой цели отдельный контекст, план, check-in и прогресс.
В чате всегда есть active_goal_id.
